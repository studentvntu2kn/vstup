Основи Docker'a

Вивчення основних понять Docker: образи, контейнери, Dockerfile та Docker Hub.

Що таке Docker образи?

Docker образ (image) – це шаблон, який містить усе необхідне для запуску контейнера. Образи можуть включати операційну систему, встановлені програми, конфігураційні файли тощо. Образи є базовою одиницею Docker і можуть зберігатися в публічних або приватних репозиторіях, таких як Docker Hub.

Основні команди для роботи з образами:

  • docker pull [image-name] – завантаження образу з репозиторію.
  • docker images – перегляд усіх доступних локально образів.
  • docker rmi [image-name] – видалення образу.

Що таке контейнер?

Контейнер – це запущений екземпляр образу, який виконує певний процес або програму. Контейнери використовують ізольоване середовище для виконання застосунків, що дозволяє уникнути конфліктів залежностей. Контейнери створюються на основі образів і мають власні файлові системи.

Основні команди для роботи з контейнерами:

  • docker run [image-name] – створення і запуск контейнера з певного образу.
  • docker ps – перегляд усіх активних контейнерів.
  • docker stop [container-id] – зупинка контейнера.
  • docker rm [container-id] – видалення контейнера.

Dockerfile

Dockerfile – це текстовий файл, що містить інструкції для створення Docker образу. У ньому можна вказати базовий образ, скопіювати файли, встановити залежності та визначити команди, які виконуватимуться при запуску контейнера. Dockerfile є ключовим інструментом для автоматизації створення образів.

Приклад Dockerfile:

# Вказуємо базовий образ
FROM python:3.9

# Копіюємо файл з кодом у контейнер
COPY app.py /app/app.py

# Встановлюємо робочу директорію
WORKDIR /app

# Встановлюємо необхідні залежності
RUN pip install -r requirements.txt

# Визначаємо команду для запуску
CMD ["python", "app.py"]

Docker Hub

Docker Hub – це публічний репозиторій образів, де користувачі можуть зберігати та завантажувати образи. Docker Hub містить тисячі готових до використання образів для різних технологій та платформ. Це зручне місце для пошуку базових образів для створення власних контейнерів.

Основні команди для роботи з Docker Hub:

  • docker pull [image-name] – завантаження образу з Docker Hub.
  • docker push [image-name] – завантаження власного образу до Docker Hub.
